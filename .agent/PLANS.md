# Execution Plans

Use an ExecPlan for work that spans multiple steps or sessions.

## Rules

- Keep each plan focused on one milestone or task.
- Include clear verification commands.
- Update progress as steps are completed.
- Do not mix web dashboard work with ESP32 firmware or Raspberry Pi service implementation in this repository.

## Skeleton

```md
# Plan: Short Name

## Goal

What this plan will accomplish.

## Scope

Files or areas expected to change.

## Out of Scope

What must not be changed in this plan.

## Steps

1. Step description.
2. Step description.

## Verification

- Command or manual check.

## Progress

- [ ] Pending item.
```
